Overview

Architecture

Projects are structured following the ITCSS guidelines. Each directory below could be considered a "layer", building upon the last. From top to bottom, the styles go from generic to explicit, far reaching to localized, and low specificity to high specificity.

|
|– settings/                    # Global variables, config switches
|   |– _defaults.scss             # Project variables and switches
|   |– _colors.scss               # Color variables
|
|– tools/                       # Default mixins and functions
|   |– _vertical-rhythm.scss      # Used by normalize to establish baseline grid
|   |– _functions.scss            # Framework functions
|   |– _mixins.scss               # Mixins to make life easier
|   |– _palettes.scss             # Generates full color palettes
|   |– _responsive.scss           # Responsive breakpoint mixin
|
|– generic/                     # Ground-zero styles
|   |– _normalize.scss            # Normalize with vertical rhythm and support-for
|
|– base/                        # Unclassed HTML elements
|   |– _typography.scss           # Base unclassed type styles
|   |– _images.scss               # Image element styles
|   |– _page.scss                 # Page level styling
|
|– objects/                     # Cosmetic-free design patterns
|   |– _grid.scss                 # Grid using Susy
|   |– _containers.scss           # Generaul use containers
|   |– _box.scss                  # Boxed content for layout
|   |– _buttons.scss              # The base button object
|   |– _forms.scss                # Form elements
|   |– _tables.scss               # Tables
|
|– components/                  # Designed components, chunks of UI
|   |– _iconic.scss               # Iconic svg sprite icons
|   |– _indicators.scss           # Indicator bubbles
|   |– _buttons.scss              # Special button components
|   |– _cards.scss                # Little cardy content capsules
|   |– _tables.scss               # Special table related components
|   |– _main-header.scss          # The main app header bar
|   |– _side-nav.scss             # The sidebar navigation
|   |– _page-header.scss          # The page header w/ actions
|   |– _page-nav.scss             # The horizontal page nav bar
|   |– _timeline.scss             # A vertical timeline
|
|– trumps/                      # Helpers and overrides
|   |– _helpers.scss              # Helper classes
|

Blockquotes

Wrap <blockquote> around any HTML as the quote. For straight quotes, we recommend a <p>.

Buttons

Buttons are used for actions, like in forms, while textual hyperlinks are used for destinations, or moving from one page to another.

Default Buttons

Use the standard—yet classy—.o-btn for form actions and primary page actions. These are used extensively around the site.

When using a <button> element, always specify a type. When using a <a> element, always add role="button" for accessibility.

Link button Large button Small button 
<button class="o-btn" type="button">Button button</button>
<a class="o-btn" href="#" role="button">Link button</a>
